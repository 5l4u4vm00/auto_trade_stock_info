# 新聞驅動台股選股策略 Skill

## 目的

協助使用者透過近一週的國內外重大新聞，有系統地分析對台灣股市的影響，並產出可操作的選股策略建議。報告以繁體中文撰寫，專有名詞（如產業術語、指標名稱、國際機構等）保留英文。

這份報告的核心價值不是「報明牌」，而是建立一套從新聞到策略的思考框架——幫助使用者理解新聞背後的投資邏輯，並據此制定自己的篩選標準。

---

## 執行流程

收到使用者請求後，依序執行以下六個步驟。每一步都很重要，不要跳過。

### Step 1：新聞搜尋

使用 web search 搜尋近一週的重大新聞，至少涵蓋以下面向：

- **台灣財經新聞**：搜尋關鍵字如「台股 本週」「台灣 經濟」「央行 政策」
- **美國/全球總經**：搜尋如「Fed interest rate」「US economy this week」「global markets」
- **產業與科技趨勢**：搜尋如「AI semiconductor news」「台積電」「電動車 供應鏈」等當前熱門主題
- **地緣政治**：搜尋如「US China trade」「geopolitics Asia」等

搜尋時注意：
- 每個面向至少搜尋 1-2 次，總共進行 5-8 次 web search
- 優先選擇可靠來源（如 Reuters、Bloomberg、經濟日報、工商時報、MoneyDJ、鉅亨網）
- 聚焦「對台股有實質影響」的新聞，過濾掉瑣碎或無關的資訊
- 如果使用者有指定特定主題或產業，優先搜尋該方向

### Step 2：新聞分類

將蒐集到的新聞歸入以下五大維度：

| 維度 | 涵蓋範圍 | 範例 |
|------|----------|------|
| 總體經濟 (Macro) | 利率、通膨、GDP、匯率、就業 | Fed 暫停升息、台灣 CPI 數據 |
| 產業政策 (Policy) | 政府補貼、法規、貿易政策、稅制 | 晶片法案補助、碳費徵收 |
| 地緣政治 (Geopolitics) | 國際衝突、外交、制裁、供應鏈風險 | 中美科技戰、中東局勢 |
| 科技趨勢 (Tech Trends) | 技術突破、新產品、產業典範轉移 | AI Agent 爆發、先進封裝需求 |
| 企業事件 (Corporate) | 財報、併購、管理層變動、訂單消息 | 台積電法說會、鴻海轉投資 |

每則新聞可以同時屬於多個維度。

### Step 3：影響方向判斷

為每則關鍵新聞標註：

- **影響方向**：利多（Bullish）/ 利空（Bearish）/ 中性（Neutral）
- **影響強度**：高（High）/ 中（Medium）/ 低（Low）
- **影響範圍**：大盤（Broad Market）/ 特定產業（Sector-specific）/ 個別公司（Company-specific）
- **時間軸**：短線（1-2 週）/ 中線（1-3 個月）/ 長線（3 個月以上）

判斷時的思考原則：
- 不要只看新聞標題，要思考二階、三階效應（例如：Fed 降息 → 資金流向新興市場 → 台股外資回流）
- 考慮市場是否已經 price in（已反映）這個消息
- 注意新聞之間的交互影響（例如：地緣政治緊張 + 科技管制 = 供應鏈轉移加速）

### Step 4：投資主題萃取

從上述新聞分析中，歸納出 2-4 個「可操作的投資主題」。

一個好的投資主題應該：
- 有明確的新聞事件作為催化劑
- 能對應到台股的具體產業鏈
- 有清晰的投資邏輯鏈條（因為 A → 所以 B → 受惠 C 產業）
- 有可衡量的觀察指標

範例格式：
```
主題：AI 伺服器需求爆發
邏輯鏈：大型科技公司資本支出上修 → AI 伺服器出貨量成長 → 台灣 ODM/零組件供應鏈受惠
對應產業：伺服器 ODM、散熱模組、高速傳輸、PCB、Power Supply
時間軸：中長線趨勢（持續 6-12 個月以上）
```

### Step 5：選股策略制定

這是報告的核心章節。針對每個投資主題，制定具體的選股策略：

**5a. 產業鏈定位**
- 列出受惠的台股產業鏈（上游/中游/下游）
- 說明哪個環節受惠最直接、彈性最大

**5b. 篩選條件建議**
針對每個主題，建議使用者可以用以下維度篩選標的：
- 產業別 / 次產業別
- 營收成長率（YoY）門檻
- 毛利率或營益率趨勢
- 本益比（P/E）或股價淨值比（P/B）合理區間
- 市值規模偏好
- 法人買賣超方向
- 其他主題相關的特殊指標

注意：篩選條件要根據主題特性調整，不要每個主題都套用相同的公式。成長型主題可以容忍較高 P/E，價值型主題則應強調低估值與高殖利率。

**5c. 風險與注意事項**
- 該主題的主要風險因素
- 可能導致策略失效的情境
- 需要持續追蹤的關鍵變數

**5d. 投資時間軸建議**
- 短線事件驅動（1-2 週）：適合什麼操作節奏
- 中線波段（1-3 個月）：觀察什麼轉折訊號
- 長線佈局（3 個月以上）：核心邏輯是否夠強

### Step 6：報告產出

將以上分析整合為一份結構化的 Markdown 報告，遵循下方的報告模板。報告完成後存為 `.md` 檔案，儲存在 `strategy/` 資料夾下，檔名格式為 `strategy/news_strategy_{YYYY-MM-DD}.md`。

## 品質要求

產出報告時，請注意以下品質標準：

- **邏輯清晰**：每個投資主題必須有完整的因果推導，不能只有結論沒有推理過程
- **務實可操作**：篩選條件要具體到使用者可以直接拿去股票篩選器使用的程度
- **平衡觀點**：不能只看利多，每個主題都必須附帶風險分析
- **時效性**：報告應反映最新一週的資訊，避免使用過時的數據
- **語言風格**：繁體中文為主，專有名詞保留英文（如 P/E Ratio、Fed、TSMC、AI、EPS）。語氣專業但不晦澀，像是一位資深分析師在跟你解釋市場

## 邊界情況處理

- 如果某一週新聞特別平淡，仍然產出報告，但可以在總結中說明「本週缺乏強烈催化劑，建議觀望」
- 如果使用者指定了特定產業（如「我只關心半導體」），則將搜尋和分析聚焦在該產業，但仍保留總經和地緣政治的影響分析
- 如果使用者問的是「現在該買什麼」，引導他理解這是策略工具而非報明牌，但仍盡力提供有價值的分析
- 永遠附上免責聲明
